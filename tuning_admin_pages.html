Тонкая настройка админки.

В одном из проектов меня не устраивала базовая функциональность админской части приложения, т.к. помимо админов доступ предоставлялся ещё и менеджерам, и рядовым сотрудникам, и следовало очень тщательно разделять выдаваемые права. Права на доступ к отдельным классам данных прекрасно регулируются встроенными инструментами, но бывают ситуации, когда надо регулировать доступ к отдельным полям (столбцам таблицы).

Базовая настройка админской панели описана в первых строках документации. Если нам требуется включить какой-то тип данных в администрируемые, достаточно использовать декоратор:

@admin.register(Driver) class DriverAdmin(admin.ModelAdmin): # ...

где Driver - это имя класса модели из файла models.py. Хороший стиль кодирования - имя класса модели входит в имя класса админки.

Далее в документации есть огромный набор функций, в назначении и обилии которых легко потеряться. Привожу примеры использования некоторых из них.

Метод def get_readonly_fields() даёт возможность настроить уровень доступа некоторым данным "только для чтения".

def get_readonly_fields(self, request, obj=None): if obj and obj.active_driver: return ['enterprise'] return []

Эта функция проверяет, существует ли объект, и заполнено ли у этого объекта поле active_driver. Если да, то смена поля enterprise невозможна, оно отображается в режиме read-only, хотя у других объектов это поле редактируемое.

По умолчанию это поле скрыто вызывается с единственной строкой: return []. В таком случае все поля доступны для редактирования.

Метод get_queryset позволяет отфильтровать данные, выдаваемые базой на запрос (queryset), т.е. выдать не полный набор данных, а лишь ту часть, которую допустимо показать пользователю. Речь идёт о данных того типа, который указан в декораторе register.

В этом проекте в каждой функции проверяется, кто сейчас онлайн. Если пользователь класса Manager, то любой объект надо отфильтровать по принадлежности к тому или иному курируемому предприятию (Enterprise).

def get_manager_enterprises(request): if Manager.objects.filter(user=request.user): mngr = Manager.objects.get(user=request.user) return mngr.enterprise.all() def get_queryset(self, request): qs = super().get_queryset(request) enterprises = get_manager_enterprises(request) if enterprises: return qs.filter(enterprise__in=enterprises) return qs

Такой изменённый queryset содержит в себе только те объекты, которые текущий менеджер может видеть (те, что относятся к курируемому предприятию).

Метод get_field_queryset позволяет отфильтровать содержимое конкретного поля. Допустим, предыдущей функцией мы отфильтровали отображаемые объекты. Пусть это будут автомобили (Vehicle). Менеджер может назначить текущего ответственного водителя на какой-то конкретный автомобиль. Но в выпадающем списке должны быть доступны только те водители, которые относятся к курируемому предприятию. Если менеджер заводит в базу новый автомобиль, то в выпадающем списке предприятий должны быть только те, с которыми работает этот менеджер, а не все предприятия вообще. Все фильтрации полей, которые в админке отображаются как выпадающий список, происходят в методе get_field_queryset:

def get_field_queryset(self, db, db_field, request): enterprises = get_manager_enterprises(request) if enterprises: if db_field.name == 'enterprise': return Enterprise.objects.filter(id__in=enterprises) if db_field.name == 'active_driver' and Manager.objects.filter(user=request.user): return Driver.objects.filter(enterprise__in=enterprises) return super().get_field_queryset(db, db_field, request)
